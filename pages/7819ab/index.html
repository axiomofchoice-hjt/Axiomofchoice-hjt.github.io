# 简介

传送门

一个解密游戏，规则由玩家发掘，难度较高。

鼠标操作，有几个关卡可用方向键。

# 攻略 1-10

# 第一关

规则

636363 个数字，玩家需要在 666 次机会里猜出正确数字。如果猜中，方块会变红色；猜的数字比正确数字小，方块会变黄色；猜的数字比正确数字大，方块会变蓝色。

（事实上正确数字不是一开始就生成好的，会根据玩家猜的数字进行调整）

攻略

每次点击答案的范围内最中间的数字，并缩小答案的范围。

比如一开始答案范围是 [1,63][1,63][1,63]，第一次点击 1+632=32\tfrac{1+63}{2}=32​2​​1+63​​=32。如果这个方块是蓝色，答案范围缩小为 [1,31][1,31][1,31]，第二次点击 1+312=16\tfrac{1+31}{2}=16​2​​1+31​​=16；如果这个方块是黄色，答案范围缩小为 [33,63][33,63][33,63]，第二次点击 33+632=48\tfrac{33+63}{2}=48​2​​33+63​​=48。

吐槽

很显然这就是二分查找算法。作为新手引导关卡当然要选择广为人知的算法了。

# 第五关

规则

和第一关比较类似。646464 个数字，玩家需要在 999 次机会里猜出最小的数字（即 000）。每次点击可以显示方块内的隐藏数字，如果猜中，方块会变红色。000 前面隐藏数字单调递减，000 后面的数字单调递增。

（同第一关，000 的位置会根据玩家猜的数字进行调整）

攻略

依次点击 40,26,16,11,7,5,3,240,26,16,11,7,5,3,240,26,16,11,7,5,3,2，然后点击 1,41,41,4 中的一个。（根据最小值的范围判断）

吐槽

作为第一关的加强版，这个关卡需要 0.618 优选法（一般的三分是不太行的），过于专业，非战斗人员建议放弃。

# 第六关

规则

点击方向键或者键盘方向键可以控制角色（显示 P 的蓝色方块），玩家需要绕过障碍物（显示正方形的橙色方块），吃到两枚金币（显示 ￥ 的黄色方块）。特别地，一个方向的按键一旦被按下，之后的两回合该按键不能被使用（显示红色）。地图的最左边可以一步走到最右边，反之亦然。

攻略

重复 555 次 (右上下)，重复 444 次 (右左下)，上左。

吐槽

也是有隐藏操作，有提示应该简单些了吧。（但是可以证明不用隐藏操作是不行的，所以之前版本完全没有提示）

# 第八关

规则

黄色方块可以让分数加一，红色方块可以让分数减一，绿色方块可以让分数乘二。玩家需要让分数恰好等于 200020002000 分

攻略

点击 222 个黄色、555 个绿色、红色、绿色、红色、444 个绿色。

吐槽

其实就是二进制。2000=(11111010000)2=((27−1)×2−1)×242000=(11111010000)_2=((2^7-1)\times2-1)\times 2^42000=(11111010000)​2​​=((2​7​​−1)×2−1)×2​4​​。

# 第九关

规则

点击左侧某个方块时，左侧除了这个方块外其他所有方块的状态反转（一共 111 和空两个状态）。玩家需要让左侧和右侧看起来完全相同。

攻略

记住一开始左侧和右侧状态相同的方块，闭着眼睛点一遍。正经人都是闭着眼的（确信）。

吐槽

哦，简单题（来自 Codeforces 玩家的评价）。本来只要让所有方格全变 111 状态即可，后来加强了。

# 第十二关

规则

点击黄色方块时，会平移某一行/列。玩家需要让绿色方块全部移动到显示五角星的区域内。

攻略

3 左，3 上，4 右，3 上，4 下
3 左，4 下，3 左，4 左，1 上

即，第 3 行向左平移，第 3 列向上平移，第 4 行向右平移，第 3 行向上平移，第 4 列向下平移，第 3 行向左平移，第 4 列向下平移，第 3 行向左平移，第 4 行向左平移，第 1 列向上平移。

吐槽

这是拼图系列第一个谜题，解法极多，大胆乱搞吧。另外留一个问题，如果初始状态和最终状态反一下该怎么玩。（感觉难多了）

# 第二十二关

规则

地图中暗藏两个宝藏，点击宝藏会显示五角星，点击非宝藏方块会显示到两个宝藏的曼哈顿距离之和，曼哈顿距离定义为横坐标之差的绝对值加纵坐标之差的绝对值。玩家需要在 12 步内找到两个宝藏。

两个宝藏可以确定一个矩形，这个矩形内所有方格的数字（如果显示的话）都是相同的，而矩形外的方格数字，会增加两倍到这个矩形的最短曼哈顿距离。利用这个规律可以用很少数字确定这么一个矩形。

算是猜数游戏、找零游戏的二维版本了。
