字节跳动 AML 面经

# 基本信息

岗位：机器学习平台研发实习生 - AML 职位

时间：

  • 2022/2/14 一面
  • 2022/2/15 二面

我是谁：一个只会算法竞赛的伞兵。

# 一面

自我介绍后，面试官让我从计组、操作系统、计网中选一个擅长的，然后回答问题。

（我看了凯爷面经，以为没有八股文，所以就没准备）果然我答得稀烂，突然想起老版简历里有“熟悉计组、操作系统、计网”的自不量力的话，于是开始求饶。

问题是：假如 client ping 了 server，TTL = 2ms，20ms 后未收到回复，原因？

（面试后想了一下感觉确实太基础了，前提是我能反应过来 ping 是什么，当时以为 ping 是 TCP 呜呜）

面试官不断感慨他的技术和我的技术太不重叠，问不了什么问题。

然后面试官问了哈希表的哈希冲突解决方法，自然是开放寻址法和拉链法二连。

然后面试官又问了这两种方法的优缺点。

啊哈哈又是我没见过的八股文了，我只好说自己没深入了解过（感觉不懂装懂的代价更大），但是面试官让我想到什么说什么。

那我只好乱说了。

我说拉链法的链表需要额外内存，申请释放这些内存有一定开销。面试官表示一般这些内存与数据相比不会很多。

我说开放寻址法不支持删除，面试官说怎么不能删除。（当时我确实不知道 hopscotch hash 这种算法）

然后我说拉链法在链表太长时可以转换成树型结构来优化，像 Java 那样。

这时面试官说这些不都是开放寻址的缺点吗，然而实际应用中用的都是开放寻址法 blabla。（害，不是让我随便说吗，而且我所知的哈希表应用只有 C++/Java，都是拉链法实现的）

最后终于开始撸代码了，面试官让我实现单链表排序，我就撸了个归并排序。在复杂度理论最优的情况下，归并排序应该是最好写的了。

面试官问我有什么要问的吗，我又开启求饶模式，2333。

# 二面

二面没自我介绍，终于不用背那段自己都觉得离谱的话了。

面试官说他今天早上就在看我的博客了。（可能代替了自我介绍？）我说献丑了。

先聊了点实习相关的事。

上来就撸代码，太幸福了。

问题是实现一个数据结构，支持添加一个整数、删除一个整数、询问中位数。

我一看，中位数，这不是经典双堆吗。然后手比脑子还快，打出 priority_queue。

面试官打断我说，堆支持删除任意整数吗？我没说不行，因为脑子里已经飘出经典双堆维护可删堆的想法了。

经典 + 经典就变成 4 个堆，虽然实现起来异常复杂但是没办法，只能继续装下去，不然白忙活了（？）。

当然写的时候发现一个不足，这个算法不支持删除不存在的整数。不过面试官放过我了。

最后面试官没有让我写测试，只是问了我可删堆的原理。然后表示很满意。

（后来我问了一下正解，开两个 set/multiset 就行了，我怎么没想到）

然后阅读了一下英文资料，大意是对一系列 32 位整数进行数据压缩，是一种变长的编码。（面试官真辛苦，又用中文给我讲了一遍，我已经看懂就没听 doge）

（我说这不就类似 utf-8 编码吗，对 unicode 的一种压缩，面试官说是也不是）

问题是给定一个数组，让我压缩成二进制数据放入字符串中。

我不仅是个老 acmer，而且对位运算也有一定研究，这种题当然没问题。面试官对我使用 __builtin_clz 的装杯行为表示赞扬。

面试官问我还有什么要问的吗。我觉得二面这么顺利也不需要求饶了，就说没有。

# 总结

得背八股、做点工程了。
