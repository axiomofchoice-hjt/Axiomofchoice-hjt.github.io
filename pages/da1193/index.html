从 C++ 到 Rust

C++ 本身确实是个很优秀的语言，奈何 C++ 历史包袱太重，要兼容 C 的很多语法，还从 C 中继承下了很多 UB（未定义行为）。

最近学了一手 Rust，也算是了解没有历史包袱的“C++”大致可以发展成什么样。

# Rust 与 C++ 的一些对比

编译器 rustc、构建工具 cargo 用起来是真的舒服，这是吸引我的一个重要原因。

Rust 的 Result 类型代替 C++ 异常处理。虽然不知道开销相比于 C++ 优化多少，但是将异常作为一个类型来约束程序员，让程序员不得不处理可能的错误，还是有可取之处的。

C++ 中的一些类型（比如 vector）赋值就是拷贝，容易不经意间产生开销。Rust 必须用 clone 来表示。

Rust 的 enum 同时具有 C++ enum、union 两个功能，既简化了代码复杂性，又保证了类型安全。

在 C++ 中的 UB 太多，这些问题在 Rust 看来都在编译期解决了。

引用，在 C++ 中对应指针，但是 Rust 有严格的引用失效机制。关于这点，有时候确实有点头疼，恨不得直接用 unsafe（比如交换 Vec 里的两个数，不过 Vec 自带了 swap 方法）。

迭代器是 Rust 中的一个比较先进的东西了，功能很强大，感觉弥补了 C++ 的一大遗憾。

可空类型 Option 必须使用模式匹配来得到值。直接避免了 C++ 等语言中忘记检测空指针引发的错误。（不过好在 C++ 也出现了 optional）

多线程中，线程间共享数据必须使用锁（Arc<Mutex<T>>），相当于 C++ 的互斥锁和临界资源绑定在了一起。
